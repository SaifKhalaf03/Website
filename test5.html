Built a Redis-Like In-Memory Data Store with C/C++

Systems Programming, Networking, Data Structures | 2024

Inspired by the inner workings of Redis, this project involved building a high-performance, in-memory key-value store from scratch using C and C++. It was an in-depth exploration of network programming, memory management, and efficient data structures.

- Designed a custom TCP server to handle client connections efficiently, processing commands in real-time.

- Implemented optimized data structures, such as a hash table, for fast key-value lookups, ensuring minimal latency.

- Enabled data persistence using Append-Only File (AOF) logging, ensuring durability and recovery of data.

- Implemented multi-threaded concurrency handling with mutexes, allowing multiple clients to interact with the data store simultaneously.

This project strengthened my understanding of how databases and caching systems operate under the hood, providing hands-on experience with low-level programming and system design.

- Clean Execution & Git-Tracked Development – Structured commits and robust system termination ensure reliability.

This project reflects my ability to build efficient, thread-safe systems at scale—ideal for roles in backend engineering, simulations, and distributed computing.
